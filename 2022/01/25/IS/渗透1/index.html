# 1： 确定目标

渗透目标常见应用
操作系统Linux 主机 /windows 主机
数据库系统SQL Server/MySql/Oracle/DB2/PostgreSQL
应用系统FTP/DNS/DHCP/WEB/RDP
网络设备防火墙 / 入侵检测系统 / 交换机 / 路由器

风险规避

  1. 明确渗透测试范围
  2. 测试之前对重要数据进行备份
  3. 不要进行 DDos/CC 攻击，不破坏数据
  4. 可以对原始系统生成镜像环境，然后对镜像环境进行测试
  5. 任何测试执行之前必须和客户进行沟通，以免引来不必要的麻烦
